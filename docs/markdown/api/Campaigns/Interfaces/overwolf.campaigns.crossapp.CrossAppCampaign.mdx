Overwolf APIs / overwolf / overwolf / campaigns / crossapp / CrossAppCampaign

Container that represent a shared data parameters.

## Properties

| Property                                                   | Type       | Description                                                                                                                                |
| ---------------------------------------------------------- | ---------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| <a id="property-action"></a> `action`                      | `string`   | The type of action this cross-app campaign supports. This is a free-text string.                                                           |
| <a id="property-data"></a> `data`                          | `any`      | Information about the cross-app campaign. This is a free-form json object that gives more instructions on the required action.             |
| <a id="property-expiration"></a> `expiration`              | `number`   | Expiration date expressed in milliseconds since epoch (Unix Time, UTC). e.g. Date.now() or (new Date()).getTime()                          |
| <a id="property-id"></a> `id`                              | `string`   | An id to identify the campaign (action/conversion). `id` should be unique per an extension (two different extensions can use the same id). |
| <a id="property-owner_app_uid"></a> `owner_app_uid?`       | `string`   | The UID of the app that owns the targeted cross-app campaign.                                                                              |
| <a id="property-target_apps_uids"></a> `target_apps_uids?` | `string`[] | An array of app UIDs this cross-app campaign targets.                                                                                      |
