# Overview

Use this API to manage Overwolf extensions (apps) — launch other extensions, query and exchange runtime info, check for updates, and react to lifecycle events such as app launch, update, and uncaught exceptions.

| API | Description |
| --- | --- |
| [current](./current/Overview) | - |
| [overwolf.extensions.ExtensionType](./Enums/overwolf.extensions.ExtensionType) | The type of an Overwolf extension. |
| [overwolf.extensions.ExtensionUpdateState](./Enums/overwolf.extensions.ExtensionUpdateState) | The update state of an extension. |
| [overwolf.extensions.checkForExtensionUpdate](./Functions/overwolf.extensions.checkForExtensionUpdate) | Checks if an update is available for the calling extension. |
| [overwolf.extensions.getExtensions](./Functions/overwolf.extensions.getExtensions) | Return service providers manifest data. |
| [overwolf.extensions.getInfo](./Functions/overwolf.extensions.getInfo) | Gets an extension's info string. |
| [overwolf.extensions.getManifest](./Functions/overwolf.extensions.getManifest) | Returns the requested extension's manifest object. |
| [overwolf.extensions.getRunningState](./Functions/overwolf.extensions.getRunningState) | Gets the running state of an extension. |
| [overwolf.extensions.getServiceConsumers](./Functions/overwolf.extensions.getServiceConsumers) | Return service providers manifest data. |
| [overwolf.extensions.launch](./Functions/overwolf.extensions.launch) | Launch an extension by its unique id. |
| [overwolf.extensions.registerInfo](./Functions/overwolf.extensions.registerInfo) | Requests info updates for extension. Will also be called when the extension launches/closes. |
| [overwolf.extensions.relaunch](./Functions/overwolf.extensions.relaunch) | The app will relaunch itself. |
| [overwolf.extensions.setInfo](./Functions/overwolf.extensions.setInfo) | Sets a string or object for other extensions to read. |
| [overwolf.extensions.unregisterInfo](./Functions/overwolf.extensions.unregisterInfo) | Stop requesting info for extension. |
| [overwolf.extensions.updateExtension](./Functions/overwolf.extensions.updateExtension) | This functions allows apps to check and perform an update without having to wait for Overwolf to do so. |
| [overwolf.extensions.AppInstallationEvent](./Interfaces/overwolf.extensions.AppInstallationEvent) | Event data fired when an extension is installed. |
| [overwolf.extensions.AppLaunchTriggeredEvent](./Interfaces/overwolf.extensions.AppLaunchTriggeredEvent) | Event data fired when the app is launched while already running. |
| [overwolf.extensions.CheckForUpdateResult](./Interfaces/overwolf.extensions.CheckForUpdateResult) | Result of `checkForExtensionUpdate`, reporting whether an update is available. |
| [overwolf.extensions.ExtensionUpdatedEvent](./Interfaces/overwolf.extensions.ExtensionUpdatedEvent) | Event data fired when the extension has been updated to a new version. |
| [overwolf.extensions.ExtensionWindowData](./Interfaces/overwolf.extensions.ExtensionWindowData) | Per-window configuration declared in the manifest `data.windows` map. |
| [overwolf.extensions.GetExtensionsResult](./Interfaces/overwolf.extensions.GetExtensionsResult) | Result of `getExtensions`, containing the list of installed extensions. |
| [overwolf.extensions.GetInfoResult](./Interfaces/overwolf.extensions.GetInfoResult) | Result of `getInfo`, containing the info string or object set by another extension. |
| [overwolf.extensions.GetManifestResult](./Interfaces/overwolf.extensions.GetManifestResult) | Result of `getManifest`, combining the base `Result` with the full `Manifest` object. |
| [overwolf.extensions.GetPhasedPercentResult](./Interfaces/overwolf.extensions.GetPhasedPercentResult) | Result containing the phased rollout percentage for the calling extension. |
| [overwolf.extensions.GetRunningStateResult](./Interfaces/overwolf.extensions.GetRunningStateResult) | Result of `getRunningState`, indicating whether the target extension is active. |
| [overwolf.extensions.Manifest](./Interfaces/overwolf.extensions.Manifest) | Representation of manifest.json |
| [overwolf.extensions.Metadata](./Interfaces/overwolf.extensions.Metadata) | App metadata as declared in the manifest `meta` block. |
| [overwolf.extensions.Point](./Interfaces/overwolf.extensions.Point) | A top/left coordinate pair used to describe a window's starting position. |
| [overwolf.extensions.ServiceProvidersDataResult](./Interfaces/overwolf.extensions.ServiceProvidersDataResult) | Result of `getServiceConsumers`, providing service-provider manifest data. |
| [overwolf.extensions.Size](./Interfaces/overwolf.extensions.Size) | A width/height pair used to describe window dimensions. |
| [overwolf.extensions.UncaughtExceptionEvent](./Interfaces/overwolf.extensions.UncaughtExceptionEvent) | Event interface for the `onUncaughtException` event, which uses a custom callback signature. |
| [overwolf.extensions.UpdateExtensionResult](./Interfaces/overwolf.extensions.UpdateExtensionResult) | Result of `updateExtension`, describing the outcome of an update attempt. |
| [overwolf.extensions.WebAppSettings](./Interfaces/overwolf.extensions.WebAppSettings) | The `data` block of the manifest, containing all app-level settings. |
| [io](./io/Overview) | Use this API to perform file system operations (read, write, copy, move, delete, list) within the sandboxed storage spaces available to an extension. |
| [shareddata](./shareddata/Overview) | Repairs the schema registration, for an extension where the manifest contains url_protocol |
| [overwolf.extensions.Permission](./Type-aliases/overwolf.extensions.Permission) | A union of all permission strings an app may request in its manifest. |
| [overwolf.extensions.UncaughtExceptionCallback](./Type-aliases/overwolf.extensions.UncaughtExceptionCallback) | The following types are related to the `onUncaughtException` event - which is a different than the usual events. |
| [overwolf.extensions.onAppLaunchTriggered](./Variables/overwolf.extensions.onAppLaunchTriggered) | Fires when the current app is launched while already running. This is useful in the case where the app has custom logic for clicking its dock button while it is already running. The event contaisn an 'origin' string which what triggered the app launch (dock, storeapi, odk, etc...) |
| [overwolf.extensions.onExtensionUpdated](./Variables/overwolf.extensions.onExtensionUpdated) | Fires when the current app's newest version has been installed. This most often means that an app relaunch is required in order for the update to apply. |
| [overwolf.extensions.onTrayIconDoubleClicked](./Variables/overwolf.extensions.onTrayIconDoubleClicked) | Fired when the tray icon is double clicked. |
