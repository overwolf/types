# Overview

APIs moved to X

| API | Description |
| --- | --- |
| [overwolf.social.twitter.getUserInfo](./overwolf.social.twitter.getUserInfo) | If the user is currently logged into Twitter, this will return userinformation:&#123; avatar: "http://abs.twimg.com/sticky/...", id: "111111111112222222" name: "full name" screenName: "screenname123"&#125; Otherwise, an error is returned. |
| [overwolf.social.twitter.performLogout](./overwolf.social.twitter.performLogout) | Performs a "strong" sign out of Twitter, so that even if the userperforms a login via the Overwolf Settings / Accounts page, he will be considered signed out. |
| [overwolf.social.twitter.performUserLogin](./overwolf.social.twitter.performUserLogin) | Opens the login dialog. There is no callback for this method and theonly way to know if the user signed in is via `onLoginStateChanged`. |
| [overwolf.social.twitter.share](./overwolf.social.twitter.share) | If the user is currently logged into Twitter, this will perform the media share (image or video). |
